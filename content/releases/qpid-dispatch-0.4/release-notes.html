Qpid Dispatch 0.4 Release Notes

Dispatch is a lightweight AMQP message router library. More about Qpid Dispatch.

For more information about this release, including download links and documentation, see the release overview.

New features and improvements

  • DISPATCH-6 - Implement link-routing
  • DISPATCH-97 - Router Engine Redesign
  • DISPATCH-100 - Accelerate the router-advert interval during topology fluctuations
  • DISPATCH-102 - Provide qdrouter schema via management interface
  • DISPATCH-108 - Identify settable, updatable and read-only attributes in management schema.
  • DISPATCH-109 - Support parsing of the Subject field from the message properties
  • DISPATCH-125 - Convert documentation to rst, generate book and HTML man pages from sphinx.
  • DISPATCH-128 - Documentation updates for 0.4

Bugs fixed

  • DISPATCH-90 - Routing table is corrupt after a router quickly restarts
  • DISPATCH-96 - Installing qpid_dispatch_site and qpid_dispatch/management does not work with "make DESTDIR"
  • DISPATCH-106 - pn link corruption after router restart
  • DISPATCH-107 - Links are not detached if the remote ends the session prematurely
  • DISPATCH-112 - Router fails with duplicate link ID errors.
  • DISPATCH-114 - SSL configuration fails to translate to proper internal names
  • DISPATCH-116 - Qpid dispatch management tools do not use SSL and SASL correctly.
  • DISPATCH-117 - SEG Fault when outgoing SSL connections fail
  • DISPATCH-118 - crash due to race on proton-c/src/error.c error structures.
  • DISPATCH-122 - Message annotations are not indexed by Symbol types
  • DISPATCH-124 - Error handling when SSL is not available.
  • DISPATCH-130 - qdrouterd with --daemon and --user options does not dump core.
