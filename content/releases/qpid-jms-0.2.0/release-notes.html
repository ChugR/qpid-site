Qpid JMS 0.2.0 Release Notes

Qpid JMS is a complete Java Message Service 1.1 client built using the Qpid Proton protocol engine.

For more information about this release, including download links and documentation, see the release overview.

New features and improvements

  • QPIDJMS-35 - allow specifying a particular alias to use when selecting a keystore entry for client auth
  • QPIDJMS-36 - ensure different link names for consumers and producers on the same Session
  • QPIDJMS-37 - upgrade to Proton 0.9.1
  • QPIDJMS-38 - updates to SSL/TLS configuration and/or handling
  • QPIDJMS-44 - Add support for dealing with Connection redirections
  • QPIDJMS-45 - add support for idle-timeout handling
  • QPIDJMS-46 - Cleanup and improve the Discovery module code
  • QPIDJMS-47 - add basic logging documentation, supply optional dep in binary for examples
  • QPIDJMS-50 - upgrade to SLF4J 1.7.12

Bugs fixed

  • QPIDJMS-33 - SASL EXTERNAL doesn't seem to work against the C++ broker
  • QPIDJMS-34 - a warning is logged when connecting for each unknown server SASL mechanism
  • QPIDJMS-39 - transport.enabledCipherSuites option seems to be ignored / not used
  • QPIDJMS-40 - When link is closed, the exceptions do not contain errors which caused the link detach
  • QPIDJMS-41 - JMS Producer needs to ensure that it settles outbound deliveries after remote
  • QPIDJMS-42 - When amqps connection fails, the program never exits?
  • QPIDJMS-43 - QueueBrowser should not participate in session transactions
